# Release checklist (Homey App Store)

## Once, before the first store release

- [ ] Replace placeholder app images (`assets/images/small|large|xlarge.png`) with real branding (250×175 / 500×350 / 1000×700). The current ones are generated by `scripts/generate-placeholders.js`.
- [ ] Replace placeholder driver images (`drivers/*/assets/images/*`, 75×75 / 500×500 / 1000×1000).
- [ ] Review driver `icon.svg` files (simple self-made shapes; replace if desired).
- [ ] Verify the app id `nl.monkeysoft.virtualdevices` is final — it cannot change after publication.
- [ ] Create the GitHub repository referenced in `.homeycompose/app.json` (`source`) or update the URL.
- [ ] Decide on a Homey Community topic and add `homeyCommunityTopicId` if wanted.
- [ ] Prepare App Store description/screenshots in the developer portal.

## Every release

- [ ] `npm run lint` passes
- [ ] `npm test` passes
- [ ] `npm run build` compiles clean
- [ ] `npm run validate` (publish level) passes
- [ ] Manual smoke test on a real Homey (see docs/testing.md)
- [ ] Version bumped in `.homeycompose/app.json` (CLI can do this during publish)
- [ ] `CHANGELOG.md` and `.homeychangelog.json` updated
- [ ] `homey app publish`
- [ ] Submit for review in the [developer dashboard](https://tools.developer.homey.app)

## Store policies to keep in mind

- No confusion with existing apps: distinct name ("MonkeySoft Virtual Devices"), own icons/assets, no copied texts.
- Privacy statement: all data stays local on the Homey; no external calls, telemetry or permissions.
